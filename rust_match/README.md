# match 표현식
1. 러스트는 match라는 제어 흐름 연산자가 있다. 사용방법은 다음과 같다.
   match <갈래에적용할값> {
        갈래(arm) => 반환값,
        .. 여러 갈래들
   }
   형태를 보면 다른 언어의 switch문과 비슷하기도 하다.
   [match]는 [if]와 다르게 꼭 [bool]타입을 쓰지 않아도 된다.
2. match 표현식 실행
   실행시 갈래에 적용할 값을 여러 갈래에 순차적으로 비교를 한다.
   해당 갈래에 여러 줄의 코드가 필요하다면 다음과 같이 사용할 수 있다.
   match <값> {
        갈래 => {
            // 코드..
            // 코드..
            1 // <-- 반환값 예시
        }
   }
3. match에 정의한 타입을 사용시
   [Option]은 Some과 None이라는 두 개의 베리언트가 있다.
   이 때 match에 Some과 None을 둘 다 정의하지 않을 경우
   러스트의 컴파일러는 둘 다 정의를 할 때까지 컴파일을 거부한다.
   러스트의 컴파일러는 이러한 버그를 허용하지 않는다.
4. other 패턴 매칭
   다른 프로그래밍언어에서 switch문의 default를 생각하면 된다.
